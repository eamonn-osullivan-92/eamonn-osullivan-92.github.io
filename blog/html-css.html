What are the differences between relative, absolute, and fixed positioning?

A look at the CSS positioning property

29 June 2022

The position property in CSS is an important one to understand if you want to have more control over the layout of your webpage. When i first started out with CSS, I ignored this property in favour of using flexbox or grid to position elements on a page. While these are both amazing and powerful features, they will only get you so far. Take a look at the definitions and differences between the key position values below.

Static:

Static is the default property. This means that the element is positioned in the normal flow of the document. If an element is static, we are unable to manipulate it's position with the top, left, right, and bottom properties.

Relative:

Relative positioning means that the element is positioned relative to itself, which might sound confusing at first. What this means is that the the element is positioned in the normal flow of the document, but unlike a static element, we can now manipulate it's position with the top, left, right, and bottom properties. For example, an element that has the following style:

position: relative;
top: 20px;
left: 20px;

Here the top: 20px; will shift the element 20px down from its standard top position. The left: 20px; will shift the element 20px to the right from its standard left position. See an example of this below; the red box is positioned in the standard document flow, whereas the blue box has the above styles applied and has shifted from it's original position.

Finaly, two other things happen when an element is positioned relatively. The first is that it allows us to manipulate the z-index of the element (i.e. setting an element on top of or below another), and secondly it effects the scope of any child elements that are positioned absolutely. I will expand on this next.

Absolute:

Absolute positioning takes the element out of the standard document flow. Once the element has been taken out of the document flow, it will either placed relative to the webpage itself, or relative to the nearest parent element that has position: relative; applied. This is a very powerful tool that allows us to position an element exactly where we want to, regardless of the position of other elements. Similar to relative positioning, we can use the top, left, right, bottom, z-index properties to acheive this. However instead of moving relative to itself, it will either move relatively to the webpage, or relative to the nearest parent element that has position: relative; applied.

Fixed:

When an element has its position value set to fixed, this means that the element is relative to the viewport. This means that the element will stay fixed to the same position regardless if you scroll up or down the page. An example can be seen at the bottom of this page. The footer below has the following code to fix it to the bottom of the screen:

.footer {
position: fixed;
bottom: 0;
left: 0;
width: 100%;

The position: fixed; takes the element out of the standard flow and positions it relative to the viewport, the bottom: 0 moves the element 0px off the bottom of the viewport (i.e. the very bottom of the screen), and the left: 0 fixes the element 0px off the left hand side of the viewport.
